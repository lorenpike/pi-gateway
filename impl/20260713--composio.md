# Composio CLI — Implementation Plan

**Date:** 2026-07-13
**Status:** Implemented

## Goal

Make the Composio CLI available in the container, with the installation under
`/opt/composio` and normal application data under the `wall-e` home directory.

## Changes

Install the latest release using Composio's installer and expose it through
`/usr/local/bin`:

```dockerfile
RUN curl -fsSL https://composio.dev/install | COMPOSIO_INSTALL_DIR=/opt/composio bash \
    && ln -s /opt/composio/composio /usr/local/bin/composio
```

`COMPOSIO_INSTALL_DIR` must be applied to `bash`, not only `curl`, because the
installer process reads it.

Do not add version pinning, PATH changes, custom cache paths, custom session
paths, or a separately installed skill. At runtime, Composio uses the existing
`HOME=/home/wall-e`; its default home data is therefore covered by the existing
persisted home volume.

Agent guidance is the Composio section in `static/APPEND_SYSTEM.md`.
Authentication can be performed later by the running agent when needed.

## Verification

After rebuilding the image:

```sh
command -v composio
composio --version
```

Expected executable: `/usr/local/bin/composio`, linked to
`/opt/composio/composio`.
